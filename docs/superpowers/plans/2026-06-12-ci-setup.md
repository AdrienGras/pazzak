# Mise en place CI + badges README — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Doter le monorepo d'une CI GitHub Actions (qualité + sécurité + couverture Codecov), d'un outillage de commit local (lefthook + commitlint gitmoji), de Dependabot (actions), de badges README, et d'une règle d'exploitation post-push.

**Architecture:** Un workflow `ci.yml` à deux jobs (`quality`, `security`) sur push `main` + PR. Couverture vitest v8 scope engine → Codecov tokenless. Hooks git via lefthook (commit-msg → commitlint ; pre-commit → biome). Versions épinglées (gel) ; la CI les verrouille via `--frozen-lockfile`.

**Tech Stack:** GitHub Actions, pnpm 11.5.3, Node 24, Vitest 4.1.8 + @vitest/coverage-v8, Codecov, lefthook, commitlint, Biome.

**Préambule toolchain (à exécuter une fois dans le shell avant toute commande pnpm) :**
```bash
export NVM_DIR="$HOME/.nvm"; . "$NVM_DIR/nvm.sh"; nvm use && corepack enable
```
Rappels : `pnpm lint` est intercepté par rtk → utiliser `command pnpm exec biome check .` en local. Branche de travail : `feat/ci` (déjà créée). Commits en gitmoji + conventional (`<emoji> type(scope): sujet`), terminés par le trailer `Co-Authored-By`.

- [ ] **Step 3 : Config commitlint — créer `commitlint.config.js`**

Accepte `<emoji> type(scope): sujet` (emoji unicode en tête, non capturé), puis applique les règles conventionnelles. Auto-suffisant (aucune lib gitmoji tierce).
```js
// Valide les messages "<emoji> type(scope): sujet" (gitmoji + conventional).
// Le headerPattern exige un emoji unicode en tête (non capturé), puis mappe
// type / scope / sujet. subject-case désactivé (sujets FR, acronymes capitalisés).
export default {
	extends: ["@commitlint/config-conventional"],
	// Laisse passer les commits de merge gitmoji ("🔀 Merge: ...") : "Merge" n'est pas un
	// type conventionnel, et le préfixe emoji court-circuite l'ignore-merge par défaut.
	ignores: [(message) => /^\S*\s*Merge:/.test(message)],
	parserPreset: {
		parserOpts: {
			// Emoji unicode en tete (selecteur de variante VS16 optionnel), non capture.
			headerPattern: /^\p{Extended_Pictographic}️?\s+(\w+)(?:\(([^)]+)\))?:\s(.+)$/u,
			headerCorrespondence: ["type", "scope", "subject"],
		},
	},
	rules: {
		"subject-case": [0],
	},
};
```

- [ ] **Step 4 : Config lefthook — créer `lefthook.yml`**

## Task 3 : Workflow CI — `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1 : Créer le workflow (jobs `quality` + `security`)**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: .nvmrc
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm check
      - run: pnpm typecheck
      - run: pnpm test:coverage
      - uses: codecov/codecov-action@v5
        with:
          files: packages/engine/coverage/lcov.info
          fail_ci_if_error: false

  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: .nvmrc
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm audit --audit-level=high
```

- [ ] **Step 4 : Valider RÉELLEMENT le workflow avec `act` (Docker)**

`act` exécute le workflow dans un conteneur émulant le runner GitHub. On cible l'event
`pull_request` (pas de filtre de branche, contrairement à `push` filtré sur `main`). On
passe l'image explicitement pour éviter le prompt interactif de premier lancement (pas de
`~/.actrc`). Le premier run télécharge l'image (~quelques min).

```bash
command act pull_request -j security -P ubuntu-latest=catthehacker/ubuntu:act-latest 2>&1 | tail -25
```
Expected: le job `security` se termine avec `✅` / `Job succeeded` (aucune vuln high+) — ou un échec `pnpm audit` à reporter (règle post-push, ne pas patcher).

```bash
command act pull_request -j quality -P ubuntu-latest=catthehacker/ubuntu:act-latest 2>&1 | tail -40
```
Expected: le job `quality` se termine avec `Job succeeded`. L'install, `pnpm check`,
`typecheck`, `test:coverage` passent ; le step Codecov peut **logger un avertissement**
(pas de token/OIDC sous act) mais ne fait **pas** échouer le job (`fail_ci_if_error:
false`). Si un step réel échoue (lint/type/test), corriger AVANT de committer.
