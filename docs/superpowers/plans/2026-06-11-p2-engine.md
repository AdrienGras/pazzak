## API boardgame.io 0.50 (vérifiée via Context7)

- Moves : `({ G, ctx, playerID, events, random }, ...args) => void | INVALID_MOVE`.
- `INVALID_MOVE`, `PlayerView` depuis `boardgame.io/core` ; types `Game`, `Move`, `Ctx` depuis `boardgame.io`.
- `random.Shuffle(arr)`, `random.Die(n)` ; `seed` sur le Game = déterminisme tests.
- Phases : `onBegin/onEnd/endIf/moves/turn/start/next`. `turn.order = { first, next }`
  (next → `undefined` finit la **phase**). `turn.onBegin/onEnd`, `turn.activePlayers`, `stages`.
- `events.endTurn/endPhase/setActivePlayers/endGame`. Game `endIf → ctx.gameover`.
- Tests headless : `Client({ game: {...PazaakGame, seed}, playerID, multiplayer: Local() })`
  depuis `boardgame.io/client` ; `client.moves.X()`, `client.events`, `client.getState() → {G, ctx}`.
- `playerView` **custom** (PlayerView.STRIP_SECRETS retire tout l'adversaire → trop agressif vs contrat §4).

## Modélisation des sets/tours (décision clé)

- 1 jeu, `numPlayers: 2`.
- **Phase `pickSideDeck`** (`start: true`) : `activePlayers {all:'pick'}`. Move `pickSideDeck(cards)`
  (10 cartes valides). `endIf` : les 2 `sideDeck` posés. `onEnd` : tire la main de 4 (`random.Shuffle`
  des 10 → 4), détermine le 1er joueur (tirage main deck, plus haute commence, égalité → retirage),
  fixe `setStarter`. `next: 'play'`.
- **Phase `play`** (boucle de sets dans une seule phase) :
  - `turn.order.first` = position de `setStarter`. `turn.order.next` = prochain joueur **actif**
    (ni standing ni busted) ; si aucun actif → on est en fin de set.
  - `turn.onBegin` : si le joueur courant est standing/busted, rien ; sinon **pioche auto** du main
    deck → board, recalcul score, détection bust / 20 (stand auto) / 9 cartes (stand auto + gagne set).
  - Moves : `playCard(handIndex, declaration?)`, `endTurn()`, `stand()`. `endTurn`/`stand` appellent
    `events.endTurn()`.
  - **Résolution de set** centralisée dans un helper `resolveSetIfOver(G, ...)` appelé quand :
    bust, 9 cartes, ou les deux joueurs `standing`. Effet : déterminer le vainqueur du set
    (cf. RULES §5), `setsWon++` (sauf tie sans Tiebreaker → set rejoué non compté), puis :
    - si `setsWon === 3` → `matchWinner`, le Game `endIf` clôt la partie (`ctx.events.endGame`).
    - sinon → vider les boards, remélanger le main deck, `currentSet++`, alterner `setStarter`,
      reset `standing/busted/playedHandCardThisTurn` ; les **mains sont conservées**.
  - Le point d'appel exact (turn.onEnd vs dans les moves) sera fixé par les tests de scénario.

## Ordre TDD (bottom-up)

1. **deck.ts** (pur) — `createMainDeck()` (40 = 4×[1..10]), catalogue side cards standard. ✅ tests directs.
2. **scoring.ts** (pur) — `scoreBoard(board)`, `isBust`, `isTwenty`, `hasNineCards`, comparaison. ✅ tests directs.
3. **types.ts** — au fil des besoins (pas de tests, support).
4. **playerView.ts** — stripping contrat §4. Test structurel (aucune valeur secrète adverse).
5. **moves.ts** + **phases.ts** — via Client headless seedé : tests de rejet par move, puis scénarios
   de parties complètes (set simple, bust, 20, 9 cartes, tie rejoué, match best-of-3).
6. **invariants.test.ts** — fast-check (RULES §7) : ≤9 cartes, score = somme, main ≤4, ≤4× chaque
   valeur, stand fige, setsWon ≤3. ≥1000 runs.
7. **ai.ts** (pur) — `(view, params) => Move`. Tests sur états donnés.
8. **index.ts** — exporte `PazaakGame`, types, `ai`.
